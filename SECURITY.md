# Security Policy

## Reporting a Vulnerability

We take security seriously at Inceptyon Labs. If you discover a security vulnerability in Unwyned or related projects, please report it responsibly to us.

### How to Report

**DO NOT** create a public GitHub issue for security vulnerabilities. Instead, please email:

ðŸ“§ **security@unwyned.com**

Please include:
- A clear description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact
- Your contact information (optional, but appreciated for follow-up)

### What to Expect

- **Initial Response**: We aim to acknowledge your report within 48 hours
- **Assessment**: We'll assess the severity and scope of the vulnerability
- **Timeline**: We'll work to patch critical vulnerabilities as quickly as possible
- **Credit**: If you wish, we'll credit your discovery in the security advisory (unless you prefer anonymity)
- **Responsible Disclosure**: We practice coordinated vulnerability disclosure and ask that you allow us reasonable time to patch before public disclosure

### Scope

This security policy applies to:
- The Unwyned mobile application (iOS & Android)
- The official Unwyned website and documentation
- Unwyned backend services and APIs

### Out of Scope

The following are generally not considered security vulnerabilities:
- Missing security headers on third-party services
- Social engineering or phishing attacks
- Physical security vulnerabilities
- Issues in third-party libraries or services (please report to the vendor)

### Security Principles

Unwyned is built with privacy and security as core principles:

- **Privacy First**: Your data is encrypted in transit and at rest
- **Minimal Collection**: We collect only what's necessary
- **No Selling**: Your data is never sold or shared with third parties for profit
- **Regular Updates**: We patch security issues promptly
- **Code Review**: Security-sensitive code undergoes thorough review

### Supported Versions

Security updates are provided for:
- **Current release**: Full support
- **Previous release**: Security patches only (90 days after new release)
- **Older releases**: No longer supported

### Security Advisories

Any significant security vulnerabilities will be disclosed via:
- GitHub Security Advisories
- Email to registered users (if applicable)
- This repository's security page
